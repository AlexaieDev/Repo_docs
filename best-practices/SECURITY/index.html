Security Policy

Reporting a Vulnerability

If you believe you’ve found a security issue, do not open a public issue. Email security@yourcompany.com or use GitHub Security Advisories to privately report it.

We aim to acknowledge within 2 business days and provide a remediation plan as soon as possible.

Supported Versions

We support the latest main branch and the most recent stable release tags.

Hardening

  • Protected branches with required reviews and status checks.
  • Secret scanning & push protection enabled.
  • Code scanning (CodeQL) and SAST in CI.
  • Short‑lived cloud creds via OIDC for deployments.
